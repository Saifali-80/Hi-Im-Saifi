==================================================
CORE PRINCIPLES
===============

I. PERFORMANCE-FIRST

* Initial render must be 2 seconds or less.
* User interactions should respond within 100ms.
* Animations must not block the main thread.
* Use requestAnimationFrame where appropriate.
* Use CSS transforms and will-change strategically.
* Lazy load all non-critical assets.
* Optimize images, fonts, and scripts.

II. MOBILE-RESPONSIVE (NON-NEGOTIABLE)

* Must function from 320px to 4K screens.
* Mobile-first CSS architecture.
* Fluid typography using clamp() and viewport units.
* Touch targets must be at least 44×44px.
* Test using real devices or browser emulation.

III. ACCESSIBILITY FIRST

* All interactive elements must be keyboard accessible.
* Semantic HTML is required.
* Every image requires meaningful alt text.
* WCAG 2.1 AA contrast compliance required.
* Icon-only buttons require aria-labels.
* Respect prefers-color-scheme by default.
* Respect prefers-reduced-motion.

IV. SELF-CONTAINED

* Core resume must function offline after first load.
* Host fonts locally.
* Vanilla HTML, CSS, and JavaScript preferred.
* External libraries must gracefully degrade.

V. SIMPLICITY & MAINTAINABILITY

* Code understandable within 10 minutes.
* Flat CSS specificity.
* Avoid unnecessary abstractions.
* Sections independently editable.
* Comment WHY, not WHAT.
* Core CSS maximum 300 lines.
* Core JavaScript maximum 200 lines.

VI. SECURITY BY DEFAULT

* HTTPS only.
* No secrets or credentials in repository.
* External links require:
  rel="noopener noreferrer"
* Implement Content Security Policy.
* Avoid inline JavaScript.
* Sanitize dynamic content.
* Prevent reverse tabnabbing.
* No high or critical security vulnerabilities.

VII. SEO & DISCOVERABILITY

Required metadata:

* Title
* Meta description
* Open Graph tags
* Twitter Card tags
* Canonical URL
* JSON-LD structured data
* robots.txt
* sitemap.xml

Targets:

* Lighthouse SEO >= 95
* Lighthouse Best Practices >= 95

VIII. PROGRESSIVE ENHANCEMENT

Core functionality must work without JavaScript.

Required without JS:

* Navigation
* Contact information
* Resume download
* Experience section
* Projects section

JavaScript may enhance:

* Theme persistence
* Animations
* Typing effects
* Skill visualizations
* Active navigation

ANIMATION GUIDELINES
====================

UI Animation:
150–300ms

Page Transitions:
400–600ms

Preferred Easing:
cubic-bezier(0.2, 0.9, 0.4, 1.1)

Rules:

* Respect reduced motion preferences.
* No infinite loops.
* Maximum 3 simultaneous animations.
* Prefer transform, opacity, filter.
* Avoid animating width, height, margin, padding, top, left.
